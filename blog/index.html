I am currently working on a project that related to Paparazzi. Surprisingly, There are only few material available online. I decide to document all my exploration here. most of the things I am writing here is mainly from my code analysis, Paparazzi wiki page and its is related Java version project Jpapabench.

Jpapabench is a Java version of Paparazzi to provide a realistic application benchmark for evaluation variouse Java Virtual Machine (JVM), especilly for Real-time JVM. Since the code base of Paparazzi have been grown dramaticly to adapt to more wild hardware driver support, it is more easier to start with Jpapbench, it mirrors the function of Paparazzi, change the cyclic execution into tasks and execute them in seperated threads. Therefore, I decide to start with Jpapabench first.

Integrate Octopress with Github Page

The most convenient parts of Octopress is its integration with Github. I can host my blog on Github pages, and edit it at anywhere as long as I have access to internet and git. On Octopress Documentation, there is an step-by-step instruction that guide you to set up connection between your local repo and remote repo.

First of all, create a git repo under github user account. To synchronize with remote repo, there is a configuration task,

The rake task will ask you for a URL of the Github repo. Copy the SSH or HTTPS URL from your newly created repository (e.g. git@github.com:username/username.github.io.git) and paste it in as a response.

Then, use rake generate to get the initialize default site at local, run rake deploy to push the site content to your repo and corresponding Github Page (e.g username.githit.io).

Repo configuration

After getting everything working, I became interested in how all of the magic happens. The answer is in the .git/config file, when we run rake setup_github_pages and provide git repo url, rake clone the git repo as remote origin and push the generated site content into repo’s master branch, then create another branch called “source” in same repo, all Octopress source code and blog markdown source code are pushed into origin source.

Thus, I can use rake generate && rake deploy to commit and push code to my blog repo’s master branch, and run git pull origin source and git push origion source to pull the my repo’s source code to any local repo, edit them and push them back to remote repo. Another takes away is rake -T will display all availible option for rake
